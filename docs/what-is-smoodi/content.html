The Smoodi Framework

Smoodi is a stand-alone and modular framework for building a WAS (Web Application Server) environment in Java.

It provides the essential foundation required to run an application, apart from the business logic, enabling developers to focus solely on business logic in a productive development environment.

1. How is Smoodi implemented?

Smoodi is a stand-aloneindependent framework. You can build a WAS application with just the Smoodi framework.

There is no need to connect it to a separate WAS engine or install other libraries individually. A single Smoodi framework dependency is all you need.


Smoodi is built using essential Java components such as the JEE API, Lombok, and Slf4j + Logback, along with a few convenience tools.

Moreover, external tools like Lombok and Slf4j are available for use outside of Smoodi as well.

2. How is Smoodi structured?

At a high level, under the umbrella project named Project Smoodi, the framework consists of several sub-frameworks:

  • WAS Engine - Physalus
  • IoC Container - Smoodi-Core
  • User Interface - Smoodi-Web

This is a modular structure composed of multiple frameworks combined together.

3. What features does Smoodi offer?

The features can be listed per framework mentioned above.

3.1 Physalus, The Engine

Physalus is the WAS engine. It handles HTTP requests by converting them into Java objects, and likewise, transforms Java objects into HTTP responses.

But that's not all. It also provides the JEE-standard FilterChain and supports parallel processing using Threads.

For more details about Physalus, refer to this article.

3.2 Smoodi-Core, the foundational framework

It offers an IoC container environment, constructor-based DI, and a Bootstrap method that initializes the entire framework.

This is the entry point and the foundation of the framework.

3.3 Smoodi-Web, the user interface for API development

This is Smoodi's User Interface for building services such as writing APIs and communicating with databases.

The Smoodi Framework

Smoodi는 Java 환경에서 WAS를 구축하기 위한 독립형, 모듈식 프레임워크입니다.

비즈니스 로직 외에 필요한 애플리케이션 실행의 기반을 제공하여 개발자가 비즈니스 로직에만 집중할 수 있는 생산성 높은 개발 환경을 지원합니다.

1. Smoodi는 어떻게 구현되어 있나요?

Smoodi는 독립형stand-alone 프레임워크입니다. Smoodi 프레임워크 하나만으로 WAS 애플리케이션을 구현할 수 있습니다.

따로 WAS 엔진을 연결하거나, 다른 라이브러리들을 따로 설치할 필요가 없습니다. Smoodi 프레임워크 의존성 하나면 충분합니다.


Smoodi는 JEE API, Lombok, Slf4j + Logback 등 기본적인 Java 환경에서 몇 개의 편의성 도구를 이용해 구현되었습니다.

또한 Lombok, Slf4j 등은 외부에서도 그대로 사용할 수 있습니다.

2. Smoodi는 어떻게 구성되어 있나요?

큰 틀을 보면, Project Smoodi라는 프로젝트 아래로

  • WAS Engine - Physalus
  • IoC Container - Smoodi-Core
  • User Interface - Smoodi-Web

등 여러 하위 프레임워크로 구성됩니다. 모듈화된 여러 프레임워크가 조합되는 모듈식 구조입니다.

3. Smoodi는 어떤 기능들이 있나요?

위에서 언급한 프레임워크별로 기능을 나열해보겠습니다.

3.1 Physalus, The Engine

Physalus는 WAS 엔진입니다. HTTP 요청을 받아 Java 객체로 만들고, 다시 Java 객체를 HTTP 응답으로 작성합니다.

이뿐만이 아닙니다. JEE 표준 기술인 FilterChain을 제공하기도 하고, Thread를 이용한 병렬 처리도 제공합니다.

Physalus에 관한 더 자세한 내용은 이 글을 참고하십시오.

3.2 Smoodi-Core, 프로젝트의 기반 프레임워크

IoC 컨테이너 환경, 생성자 DI, 그리고 프레임워크 전체를 하나의 메소드로 실행시키는 BootStrap을 제공합니다.

프레임워크의 진입점이자, 기반이 됩니다.

3.3 Smoodi-Web, API 작성을 위한 유저 인터페이스

실제로 API를 작성하고, 데이터베이스와 통신하는 등 서비스를 만들기 위한, Smoodi의 User Interface입니다.
