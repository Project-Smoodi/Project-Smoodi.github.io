The current Smoodi framework is incomplete.

The framework is functional, but contains many bugs and is still under development.

Adding Smoodi Dependencies

To add Smoodi to your project, include the following dependency in your dependencies block.

Replace 'version' with the desired Smoodi version.


dependencies {
    implementation 'org.smoodi.web:smoodi-web:version'
}
        

dependencies {
    implementation("org.smoodi.web:smoodi-web:version")
}
        

<dependency>
    <groupId>org.smoodi.web</groupId>
    <artifactId>smoodi-web</artifactId>
    <version>version</version>
</dependency>

Configuring the Smoodi Repository

Smoodi is distributed via Maven Central.

Configure the repository to your repositories block.


repositories {
    mavenCentral() // Most Java (JVM) projects, including Smoodi, are distributed via Maven Central.
                   // IDE-created projects often include this configuration by default.
}
        

repositories {
    mavenCentral() // Most Java (JVM) projects, including Smoodi, are distributed via Maven Central.
                   // IDE-created projects often include this configuration by default.
}
        

<repositories>
    <repository>
        <id>central</id> <!-- Most Java (JVM) projects, including Smoodi, are distributed via Maven Central. -->
        <url>https://repo.maven.apache.org/maven2</url> <!-- IDE-created projects often include this configuration by default. -->
    </repository>
</repositories>

Creating the Main Class

Create the main class to start the framework as shown below.


public class MainClass {
    public static void main(String[] args) {
        SmoodiFramework.startSmoodi(MainClass.class);
    }
}
    

class MainClass {
}
fun main() { // In Kotlin, the main function is independent of the class, but a main class is still needed for technical reasons.
    SmoodiFramework.startSmoodi(MainClass::class.java)
}

Running the Project

Now, run the project. You can run it from your IDE or use the terminal with the command ./gradlew run or mvn exec:java.

If it runs successfully, you should see logs similar to the following. The actual logs will be longer and more complex.

현재 Smoodi 프레임워크는 완성된 상태는 아닙니다.

프레임워크는 동작 가능하지만, 많은 오류가 있으며 계속 개발해나가는 중입니다.

Smoodi 의존성 설정

dependencies 블록에 아래와 같이 Smoodi 의존성을 추가합니다. version 위치에는 프로젝트에서 요구하는 Smoodi 버전을 입력합니다.


dependencies {
    implementation 'org.smoodi.web:smoodi-web:version'
}
        

dependencies {
    implementation("org.smoodi.web:smoodi-web:version")
}
        

<dependency>
    <groupId>org.smoodi.web</groupId>
    <artifactId>smoodi-web</artifactId>
    <version>version</version>
</dependency>

Smoodi Repository 추가

Smoodi는 Maven Central에 배포되어 있습니다.

아래와 같이 Smoodi의 Maven Central을 repositories 블록에 추가합니다.


repositories {
    mavenCentral() // Smoodi를 포함해, 대부분의 Java(JVM) 프로젝트는 Maven Central을 통해 배포됩니다.
                   // IDE로 만든 프로젝트는 기본적으로 이 구성을 포함하기도 합니다.
}
        

repositories {
    mavenCentral() // Smoodi를 포함해, 대부분의 Java(JVM) 프로젝트는 Maven Central을 통해 배포됩니다.
                   // IDE로 만든 프로젝트는 기본적으로 이 구성을 포함하기도 합니다.
}
        

<repositories>
    <repository>
        <id>central</id> <!-- Smoodi를 포함해, 대부분의 Java(JVM) 프로젝트는 Maven Central을 통해 배포됩니다. -->
        <url>https://repo.maven.apache.org/maven2</url> <!-- IDE로 만든 프로젝트는 이러한 구성을 기본적으로 포함하기도 합니다. -->
    </repository>
</repositories>

메인 클래스 작성

아래와 같이 프레임워크를 시작하기 위한 메인 클래스를 작성합니다.


public class MainClass {
    public static void main(String[] args) {
        SmoodiFramework.startSmoodi(MainClass.class);
    }
}
        

class MainClass {
}

fun main() { // Kotlin에서는 main 함수가 클래스와 독립적이지만, 기술적인 이유로 메인 클래스는 여전히 필요합니다.
    SmoodiFramework.startSmoodi(MainClass::class.java)
}
