1. Module

모듈Module이란 Smoodi에서 인지하고 있는, 정확히는 IoC 컨테이너에 등록되어 있는 객체를 말합니다. 이 모듈에 대한 Smoodi의 다양한 기능을 알아봅시다.

더 정확한 정의

객체가 생성되고 파괴될 때까지를 객체 생명주기Lifecycle라고 부릅니다. 이 생명주기를 사용자가 관리하는 게 아닌, Smoodi가 대신 관리해주는 객체를 말합니다.

사용자는 그저 클래스를 정의하고 어노테이션을 부착하면, Smoodi가 자동으로 모듈로 등록합니다. 이 작업은 모듈 로더ModuleLoader에 의해 일어납니다.

2. ModuleLoader

Smoodi에서 모듈 기능을 지원하기 위한 핵심입니다. 사용자 혹은 Smoodi의 모듈을 동적으로 불러오고, 인스턴스화합니다.

2.1 기본 동작 방식

기초적인 동작 방식은 어렵지 않습니다.

  1. 정의된 클래스들을 불러옵니다.
  2. Reflection API를 통해 @Module 어노테이션을 찾습니다.
  3. 어노테이션이 있을 경우 모듈 초기화 생성자ModuleInitConstructor를 찾고 초기화합니다.
  4. 생성된 인스턴스를 IoC 컨테이너에 등록합니다.

2.2 @Module 어노테이션

클래스에만 부착할 수 있으며, 모듈로써 등록함을 의미합니다. 위에서 언급했듯 ModuleLoader에 의해 관리됩니다.

세 가지의 매개변수가 존재하며, 대부분의 경우는 기본값으로도 충분하니 따로 설정할 필요는 없습니다.

boolean IoC default true

사용자가 별다른 동작을 하지 않아도 자동으로 IoC 컨테이너에 등록할지의 여부를 설정합니다.

모듈의 생성과 등록을 ModuleLoader에 맡기지 않고, 사용자가 직접 IoC 컨테이너를 참조해 객체를 생성 및 등록하는 경우 해당 옵션을 false로 지정합니다. 이러한 경우는 정적static 모듈이라 호칭합니다.

모듈을 IoC 컨테이너에 저장할 때엔 해당 모듈의 타입에 @Module 어노테이션이 부착되어있어야만 합니다.

byte order default 0
